
Типы машин:
  Сервер
  Терминал

) Получение информации из журналов об ошибках и работы системы
Типы журналов:
  журнал ОС
    Путь к журналу: /var/log/messages
  журнал СУБД
    Путь к журналу: /var/lib/pgsql/data/pg_log/postgresql-<день_недели>.log

Наличие журналов на разных типах машин:
  для типа Сервер
    журнал ОС
    журнал СУБД
  для типа Терминал
    журнал ОС

Инструкция по открытию журналов:
Для доступа к журналу с произвольной машины включенной в сеть откройте
терминал и выполните:
  ssh root@<ip_машины> cat <путь_к_журналу> | more
  Примечание:
    При необходимости введите "yes" на вопрос о желание продолжения соединения
    При необходимости введите пароль от root

Более подробную информацию можно найти в документации по ОС и СУБД.

) Востановление базы данных(БД) из резервной копии
Директория по умолчания для создания авто-бэкапов СУБД: /var/lib/autobackup

Для востановления БД из бэкапа выполните:
)) Зайдите на Сервер за администратора 1С
)) Если информационная база 1С не созданна на сервере:
))) Создайте информационную базу
)) Оключите 1С на время востановления из бэкапа
)) Зайдите на Сервер за администратора СУБД
)) Выполните; 
$ pgsql <имя_БД> -f <путь_к_бэкапу>
Пример:
$ psql kitchen -f /var/lib/autobackup/20150509_kitchen_d.dump
